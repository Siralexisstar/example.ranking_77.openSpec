## ADDED Requirements

### Requirement: Internal operational homepage
The system SHALL present the main homepage as an internal operational dashboard for a single box rather than as a commercial or marketing landing page.

#### Scenario: User opens the homepage
- **WHEN** a user accesses the main page
- **THEN** the page communicates box activity, rankings, and daily utility instead of promotional sales messaging
- **AND** the overall hierarchy emphasizes internal use by athletes and coaches

### Requirement: Participation summary at the top of the dashboard
The system SHALL show a top-of-page summary that helps users understand ranking participation at a glance.

#### Scenario: User scans the first section of the dashboard
- **WHEN** the top section is displayed
- **THEN** the page shows the total number of ranked athletes
- **AND** the page shows additional summary facts relevant to current box activity

### Requirement: Fixed internal section sequence
The system SHALL organize the homepage in a consistent internal workflow order so users can predict where each core area appears.

#### Scenario: User scrolls through the homepage
- **WHEN** the full dashboard is rendered
- **THEN** the weekly training overview appears before the athlete ranking list
- **AND** the join-the-ranking section appears after the athlete ranking list
